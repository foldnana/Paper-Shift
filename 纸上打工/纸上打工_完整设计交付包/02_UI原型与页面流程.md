# 《纸上打工》UI 原型与页面流程

## 一、UI 风格目标

当前 UI 原型严格参考《纸上谈亲》的视觉语言：

- 微信小游戏顶部白色栏。
- 大面积浅蓝背景。
- 圆角白色纸片卡。
- 厚描边按钮。
- 蓝色稀有信息条。
- 票据样式标签。
- 半透明遮罩弹窗。
- 底部日历与主操作按钮。
- 多代人物头像树。

## 二、当前原型文件

`纸上打工_UI原型.html`

该文件是一个静态网页原型，不依赖外部资源，直接打开即可查看。

## 三、页面清单

### 1. 创建劳动者

用途：生成角色开局。

展示：

- 时代选择
- 性别
- 年龄
- 体魄
- 识字
- 逻辑
- 社交
- 学历
- 家境
- 优势
- 资产

注意：

- 创建阶段不显示职业和工资。
- 职业必须通过求职获得。

### 2. 选择标签

用途：选择 3 个人生标签。

标签影响：

- 面试概率
- 岗位类型
- 事件选项
- 打工压力
- 退休结果
- 下一代属性

### 3. 编辑简历

用途：包装角色求职信息。

核心 UI：

- 求职意向
- 简历条目
- 显示/隐藏/夸大/无中生有
- 简历包装风险

求职意向会影响刷到的岗位。

### 4. 求职

用途：寻找工作机会。

结构：

- 上方：玩家求职者卡。
- 下方：面试岗位卡。

按钮：

- `面试第2轮`
- `再投一家`
- `就这份了`

### 5. 面试失败

用途：展示面试失败结果。

表现：

- 岗位卡灰掉。
- 灰色横幅提示失败原因。
- 底部按钮变为 `再投一家`。

### 6. 打工

用途：入职后的主循环。

结构：

- 上方：玩家当前状态。
- 下方：当前岗位状态。

按钮：

- `再干一月`
- `分配`

### 7. 工资分配

用途：分配工资，并影响家庭与下一代。

条目：

- 饮食
- 住房
- 恋爱
- 教育
- 存款

即时反馈：

- 恋爱事件概率
- 结婚事件概率
- 生子事件概率
- 后代成长加成

### 8. 新闻弹窗

用途：职业、时代、社会变化的弹窗提示。

规则：

- 不做独立页面。
- 盖在当前页面上。
- 选项不超过 3 个。

示例：

```text
新闻：
新职业「人机团队协调员」出现了。

选项：
投相关岗位
先干当前工作
攒钱观望
```

### 9. 退休结算

用途：一代结束，选择下一代。

展示：

- 退场原因
- 工作年限
- 最终职业
- 留下存款
- 精神状态
- 后代
- 退休财产分配

按钮：

- `退休完成，开启下一代`

## 四、主流程

```text
创建劳动者
 → 选择标签
 → 编辑简历
 → 求职面试
 → 面试失败 / 再投一家
 → 入职打工
 → 再干一月
 → 工资分配
 → 随机家庭/职业/时代事件
 → 退休结算
 → 开启下一代
```

## 五、交互原则

1. 每屏只做一个核心决策。
2. 每次点击都要有数值或叙事反馈。
3. 文案短，尽量像事件弹窗，而不是系统说明。
4. 失败也要给故事，不只是扣分。
5. UI 保留纸片感，不做现代职场 App 风。
